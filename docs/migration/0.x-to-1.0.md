# Migrating from 0.x to 1.0

The earlier repository packages were private milestone surfaces at version
`0.0.0`; they had no supported registry compatibility contract. Import only
documented package roots in 1.0. Replace source-private runtime imports with
`@rendered-motion/player-web` or the high-level `@rendered-motion/element`.

The stable element tag is `rendered-motion`. Register through
`defineRenderedMotionElement()` or opt into the browser-only `/auto` entry.
Authored state names remain data and are requested through `state`, `setState()`,
or `send()`.

Existing valid wire `0.1` assets and project `0.2` files remain in those version
spaces. Recompile with the final compiler when you need final provenance and
quality reports; do not assume a private milestone tarball has final package
contents.
