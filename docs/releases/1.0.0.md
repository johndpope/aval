# AVAL 1.0.0

Status: release candidate engineering implemented; publication and named-profile
certification remain protected external gates.

The release synchronizes `@pixel-point/aval-graph`, `@pixel-point/aval-format`,
`@pixel-point/aval-compiler`, `@pixel-point/aval-player-web`, and
`@pixel-point/aval-element` at `1.0.0`. It launches exact project, wire, and
build-report contracts at `1.0`; there are no pre-release compatibility
parsers or aliases.

Highlights include user-defined state graphs, continuous decoded partial loops,
packed alpha, separate H.264/H.265/VP9/AV1 assets, AV1 10-bit output, slow
compression controls, ordered HTML-like child sources, exact WebCodecs probes,
bounded range/integrity loading, page-wide resource arbitration, and an
SSR-safe custom element.

Known boundaries:

- web desktop only; no native/mobile release;
- exact H.264/H.265/VP9/AV1 WebCodecs and WebGL behavior is capability-probed;
- unsupported animation leaves optional host fallback markup visible;
- runtime scheduling reports do not prove physical display scan-out;
- observed-display evidence is not measured until a separate report passes;
- FFmpeg and codec/content licensing remain external obligations.
